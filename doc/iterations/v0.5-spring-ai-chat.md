# v0.5 Spring AI 基础对话 + SSE 流式

> **本步目标**：接入 Spring AI，按用户 API 配置调用模型；SSE 流式返回；写入 assistant 消息。  
> **预计时间**：4–5 小时

---

## 前置条件

- [ ] 完成 [v0.4](./v0.4-conversation.md)
- [ ] 用户已配置有效 API Key

---

## 依赖升级（本步）

在 `agent-server/pom.xml` 增加：

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-openai</artifactId>
</dependency>
```

> 若 Spring AI 2.0 需 Boot 4.x，可先使用 Spring AI 1.0 + Boot 3.4；升级路径写在本文末尾。

`application-local.yml` 配置默认 LLM（可被用户 DB 配置覆盖）。

---

## 实现步骤

### 步骤 1：动态 ChatModel

根据当前登录用户的 `user_api_configs` 构建：

```java
OpenAiApi api = OpenAiApi.builder()
    .baseUrl(cfg.getBaseUrl())
    .apiKey(cfg.getApiKey())
    .build();
ChatModel chatModel = OpenAiChatModel.builder().openAiApi(api).build();
```

### 步骤 2：ChatClient

```java
ChatClient chatClient = ChatClient.builder(chatModel)
    .defaultAdvisors(new MessageChatMemoryAdvisor(chatMemory))
    .build();
```

历史消息从 `messages` 表加载。

### 步骤 3：SSE 流式接口

`GET /api/chat/stream?conversationId=&prompt=`

使用 `SseEmitter` 或 WebFlux `Flux<ServerSentEvent<String>>` 推送 token 片段。

### 步骤 4：持久化

流结束后：

1. 保存 user 消息（若未存）
2. 拼接完整 assistant 回复写入 `messages`
3. 记录 `token_usage`（若 API 返回 usage）

### 步骤 5：前端

Vue 使用 `EventSource` 或 `fetch` 读 SSE，逐字显示。

未配置 API 时禁用发送按钮（对标 Python Demo）。
