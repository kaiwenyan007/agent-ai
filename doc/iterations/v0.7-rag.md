## 设计原则（对标 Python 最新版）

- **仅用户自配目录**，不索引项目 `knowledge/`
- **仅 `.md` 文件**
- 向量库目录：`.chroma/users/{userId}/`（项目根或配置项指定）
- **懒加载**：登录后不加载模型；首次 RAG 或 REBUILD 才加载

## 中间件

| 组件 | 安装方式 |
|------|----------|
| Chroma | **无需单独安装**，Spring AI Chroma starter，数据落盘 |
| Embedding | 推荐 **API**（快）；可选 Ollama 本地 |

`application.yml` 增加：

```yaml
agent:
  rag:
    chroma-base-path: ./.chroma
    chunk-size: 500
    chunk-overlap: 50
    rebuild-batch-size: 32
```

## 实现步骤

### 步骤 1：知识库配置接口

| 方法 | 路径 |
|------|------|
| GET/PUT | `/api/knowledge/config` |
| POST | `/api/knowledge/scan` |
| POST | `/api/knowledge/rebuild` |
| GET | `/api/knowledge/rebuild/progress`（SSE） |

### 步骤 2：扫描 md

```java
List<File> files = FileUtil.loopFiles(dir,
    f -> StrUtil.endWithIgnoreCase(f.getName(), ".md"));
```

### 步骤 3：切分 + 向量化

1. 文本切分（500/50）
2. `VectorStore.add` 分批写入，每批上报进度
3. 更新 `doc_count`、`chunk_count`、`last_indexed_at`

### 步骤 4：进度阶段（对标 Python）

1. 清空旧索引 5%
2. 扫描 md 10%
3. 切分 N 片段 20%
4. 加载 Embedding 25%→30%
5. 向量化 x/N 30%→90%
6. 写入 95%
7. 完成 100%

### 步骤 5：检索

`VectorStore.similaritySearch(query, topK)`，写入 `rag_queries` 日志。

### 步骤 6：Vue KNOWLEDGE 页

- 路径输入 + 选择文件夹（Windows 可后续 JNI/桥接）
- SAVE / SCAN / REBUILD
- 进度条
